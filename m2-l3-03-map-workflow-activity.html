Activity Deep Dive: Map the Workflow — Where Should AI Assist?

The slide is flagged for more detail. Here's the full activity: a 5-step workflow, 10 AI tools described on information cards, and an interactive exercise to place each tool at the right stage — with explanations for every answer.

How This Activity Works

A steel fabrication plant is deploying 10 AI-powered tools across its maintenance operation. Your job is to read each tool's description and decide which stage of the workflow it belongs in. Then use the interactive exercise to test your placements — and read the detailed explanations for every tool.

The five stages are: Data Collection → Analysis → Decision-Making → Execution → Feedback. Some tools clearly belong to one stage. Others might feel like they could go in multiple places — the explanations will show you why one stage is the primary fit.

Step 1: Read the Tool Descriptions

Tool A
Vibration Sensor Network
300 wireless vibration sensors installed on rotating equipment throughout the plant, streaming readings every 500ms to the central monitoring platform.
Tool B
Root Cause Recommender
AI system that analyzes multi-sensor patterns and generates a ranked list of probable causes with confidence scores for any flagged anomaly.
Tool C
Digital SOP Library
Tablet-accessible database of step-by-step repair procedures for every machine in the facility, with embedded safety checklists and torque specifications.
Tool D
Work Order Learning Engine
AI that analyzes all closed work orders to identify which AI diagnoses were confirmed vs. overridden, and updates the recommender's confidence models accordingly.
Tool E
Thermal Imaging System
Infrared cameras mounted at key locations that continuously capture thermal images of electrical panels and motor housings, flagging hot spots above set thresholds.
Tool F
Maintenance Schedule Optimizer
AI that analyzes machine health scores, production schedules, and technician availability to suggest the optimal timing and sequence for upcoming maintenance tasks.
Tool G
Alert Priority Ranker
AI that receives all active alerts across the plant and ranks them by urgency based on machine criticality, failure probability, and production impact — updated every 15 minutes.
Tool H
Parts Usage Predictor
AI that analyzes historical repair data and current machine health trends to forecast which replacement parts will be needed over the next 90 days, triggering inventory orders automatically.
Tool I
Recurring Failure Detector
AI that scans completed work orders looking for patterns — machines with the same failure appearing more than twice in 6 months, or the same technician consistently overriding the AI's diagnosis for a specific machine type.
Tool J
Repair Time Tracker
Mobile app that technicians use to log work order progress in real time — what steps have been completed, what parts were used, and how long each task took.

Step 3: Full Explanations for Every Tool

✅ Answer Key with Explanations
Why Each Tool Belongs Where It Does
Data Collection
Tool A — Vibration Sensor NetworkTool E — Thermal Imaging System
Tool A is a classic data collection tool — sensors that physically measure and stream raw readings. They generate data; they don't interpret it. The 300-sensor network covering the whole plant is only possible because sensors are cheap and AI can monitor all 300 simultaneously. Tool E (thermal cameras) also belongs here — it's another type of sensor, just capturing thermal images rather than numeric readings. Both tools gather raw environmental data without making any interpretation about what it means.
Analysis
Tool B — Root Cause RecommenderTool G — Alert Priority Ranker
Tool B takes raw sensor data and interprets it — pattern-matching against historical failure signatures to generate probable causes with confidence scores. This is analysis: turning numbers into meaning. Tool G might seem like Decision-Making since it ranks alerts, but its primary function is analytical — it processes incoming alert data and applies priority logic to rank them. The output of Tool G is information (a prioritized list) not a decision. A human still decides what to do with that ranked list.
Decision-Making
Tool F — Maintenance Schedule OptimizerTool H — Parts Usage Predictor
Tool F goes beyond analysis into recommendation — it generates an optimal maintenance schedule (a course of action) rather than just interpreting data. This is the border between Analysis and Decision-Making: the optimizer is proposing what to do and when. A human reviews and approves the schedule before it becomes official. Tool H (parts predictor) triggers inventory orders — that's a recommended action, not just analysis. It sits in Decision-Making because its output influences procurement decisions, even if those decisions are reviewed by a human buyer.
Execution
Tool C — Digital SOP LibraryTool J — Repair Time Tracker
Tool C is used during the physical repair process — the technician references it at the machine while doing the work. It's an execution support tool, not an analysis or planning tool. It doesn't collect data or make decisions; it supports the human performing the task. Tool J (repair time tracker) is also active during execution — technicians use it in real time as they work to log progress and parts usage. Both tools are embedded in the execution stage, not before or after it.
Feedback
Tool D — Work Order Learning EngineTool I — Recurring Failure Detector
Tool D is the purest feedback tool — its entire purpose is to analyze completed work orders and use that data to improve the AI's future recommendations. It closes the loop. Tool I (recurring failure detector) also belongs in Feedback — it scans historical data to find patterns across many events over time. Its outputs don't influence a single repair; they improve the system's understanding of recurring problems. Both tools are backward-looking (what happened) in service of forward improvement (what to do differently).

📌 Activity Summary

What This Activity Teaches

AI tools cluster in the Data Collection and Analysis stages — where the advantage of processing speed and scale makes AI uniquely powerful. Monitoring 300 sensors simultaneously or pattern-matching across thousands of historical failures are tasks no human team can replicate at that scale.

The border between Analysis and Decision-Making is "does the output recommend an action?" Analysis produces information. Decision-Making produces courses of action — and those must always be reviewed by humans before becoming actual decisions.

Execution stage AI tools support humans doing physical work — they don't replace it. Digital SOPs and time trackers make technicians more efficient, not unnecessary. All physical maintenance work remains human work.

Feedback stage tools are how the whole system improves over time. They work backward from completed events to improve future performance. Without thorough documentation from humans in the Execution stage, Feedback tools have nothing to learn from.
